## 工夫した点

### 1. タグ管理機能

ログには複数のタグを設定でき、タグをクリックすることで関連する研究ログを素早く検索できます。

また、ログを削除した際には未使用となったタグも自動で削除するように実装し、タグ一覧が不要なタグで増え続けないように工夫しました。

### 2. 検索機能

キーワード検索とタグ検索を独立した状態として管理し、両者を組み合わせた複合検索を実装しました。

検索処理にはSpring Data JPAのJPQLを利用し、タイトル・本文検索とタグ検索を柔軟に組み合わせられるよう設計しています。

### 3. 保守性を意識した設計

バックエンドではController・Service・Repositoryに責務を分離し、データベース操作や業務ロジックを適切に分割しました。

フロントエンドでは役割ごとにコンポーネントを分割し、共通スタイルをcommon.cssにまとめることで保守性と再利用性を高めています。

### 4. UI/UXの改善

日常的に研究ログを記録、編集することを想定し、フォームの開閉アニメーション、編集時の自動スクロール、ログの折りたたみ表示など、快適に操作できるUIを意識して設計しました。

## 制作目的

本プロジェクトは、研究を行う身近な学生に利用してもらうことを想定した研究ログアプリケーションです。

開発にあたっては、React・Spring Boot・PostgreSQLを用いたフルスタック開発を経験することだけでなく、実際の利用者から継続的にフィードバックを受けながら機能改善やリファクタリングを行う開発プロセスを経験することを重視しています。

ゲーム開発においても、ユーザーの意見をもとに機能やUI/UXを改善し続けることが重要であると考えています。本プロジェクトでは、研究室という身近な環境で実際の利用者の声を取り入れながらシステムを改善する経験を積み、その知見をゲーム開発へ活かすことを目的としています。

現在も継続的に機能追加やリファクタリングを行っており、Dockerによる環境構築やAWSへのデプロイなどを通して、より実用的なアプリケーションへ発展させていく予定です。

## AIの利用について

本プロジェクトでは、生成AI(ChatGPT)を開発支援ツールとして積極的に活用しました。

具体的には、

- 要件定義や機能設計の壁打ち
- ロゴの生成
- UI/UXの改善案の検討
- リファクタリング方針の相談
- React・Spring Bootなど、初めて触れる領域の学習
- コードレビューや設計上の改善点の確認
- READMEの作成支援

などに活用しています。

一方で、実装内容や設計方針に関してAIに丸投げするのではなく、あくまで「設計・レビュー・学習を支援する開発パートナー」として活用しました。最終的には「AIを利用せずとも制作できる力はあるが、効率や正確性のためにAIを利用する」という状態を目指して常に自分で理解できてから進めるということを心がけました。
